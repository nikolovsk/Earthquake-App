# Документација за Earthquake Monitoring Application 

### 1. Опис на проектот
Темата на овој проект е имплементација на систем за следење и визуелизација на земјотреси со користење на податоци од надворешно API.
Идејата на проектот е развој на full-stack веб апликација која овозможува автоматско преземање, обработка, филтрирање и 
приказ на податоци за земјотреси од јавниот USGS Earthquake API (https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_hour.geojson).  
Апликацијата овозможува корисникот да добие реални, ажурирани податоци за земјотреси, да ги филтрира според јачина (magnitude) 
и време на настанување, како и да ги прегледува во табеларен и географски приказ. 
Дополнително, овозможува и бришење на поединечни записи и автоматско освежување на податоците од надворешниот API.  
Backend системот е имплементиран со Java Spring Boot и користи PostgreSQL база за чување на обработените податоци. 
Frontend делот е изграден со React + TypeScript и овозможува интерактивен UI.

### 2. Имплементирани функционалности 
- Преземање на податоци од надворешен API: Backend системот автоматски презема податоци за земјотреси со помош на WebClient, ги парсира
и ги извлекува само релевантните полиња (magnitude, magType, place, title, time).  
- Перзистенција на податоци: Обработените податоци се чуваат во PostgreSQL база на податоци.
Пред секое ново освежување, постоечките податоци се бришат и се додаваат нови ажурирани податоци во базата на податоци.
- Логика за филтрирање: Системот овозможува филтрирање на земјотреси според минимална јачина (magnitude) и време на настанување.
- Бришење на поединечни земјотреси по id.  
- Табеларен приказ: Имплементиран е React интерфејс кој прикажува податоци во табеларна форма и овозможува интеракција со системот
(филтрирање и бришење).  
- Географска визуелизација на земјотреси: Системот овозможува и приказ на земјотресите на интерактивна мапа
со користење на Leaflet и React-Leaflet библиотеката. Секој земјотрес е прикажан како marker на мапата според неговите координати
(latitude и longitude), што овозможува подобра визуелна анализа на географската распределба на настаните. Со клик на marker на мапата
се прикажуваат детали за конкретен земјотрес.  
- REST API архитектура -> Имплементирани се чисти REST ендпоинти за преземање и обработка на податоците за земјотреси.
- Layered архитектура на backend: Апликацијата е организирана во слоеви со јасна поделба на одговорности.  
- MapStruct за DTO мапирање -> Се користи MapStruct за автоматска конверзија помеѓу Entity и DTO објекти, што го подобрува квалитетот и
одржливоста на кодот.
- Подобрено справување со грешки -> Имплементирано е справување со грешки и исклучоци во frontend и backend, вклучувајќи loading состојби, 
приказ на error пораки, како и стабилна комуникација со API.

### 4. Тестирање
Имплементирани се и unit тестови за service layer со користење на Mockito.  
Тестовите покриваат:  
- refresh на податоци од API  
- бришење на земјотреси  
- филтрирање на податоци според јачина и време  

### 5. Заклучок
Системот претставува едноставна, но функционална апликација за обработка и приказ на реални податоци за земјотреси, со цел демонстрација на 
интеграција помеѓу frontend, backend и база на податоци, како и работа со надворешни API сервиси.
Решението е дизајнирано со фокус на чист код, добра архитектура и одржливост.
